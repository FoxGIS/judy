路径

所有API部署到以下基础路径：

http://judy.com/api/v1

access_token

除用户注册和用户登录外，所有的API都需要access_token获取权限。 请妥善保管好您的access_token，因为access_token可以获取到您的所有资源权限。 设置access_token有四种方式，优先级从上至下有：

  1. url方式(不推荐)
  1. body方式
  1. cookie方式
  1. header方式
GET /{endpoint}?access_token={your_access_token}

用户注册

用户注册不需要提供access_token

POST
/users

请求示例

{
  "username": "jingsam",
  "password": "123456"
}

响应成功

{
  "username": "jingsam",
  "access_token": "{your_access_token}",
  "updatedAt": "2016-05-06T12:33:02.689Z",
  "createdAt": "2016-05-06T11:33:02.689Z",
  "is_verified": false
}

响应失败

{
  "error": "密码长度太短"
}

用户登录

用户登录不需要access_token，但登录成功后会生成新的access_token，已有的access_token会失效。
